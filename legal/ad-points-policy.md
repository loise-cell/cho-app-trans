# 廣告與點數規範

最後更新日期：2026-07-11

本文件說明 Cho App Trans 的獎勵廣告與點數機制。

## 1. 點數取得

| 項目 | 規則 |
|------|------|
| 獎勵廣告 | 完整觀看 Google AdMob 獎勵廣告後發放點數 |
| 一般獎勵 | +1 點（預設） |
| 幸運爆擊 | 1% 機率 +10 點 |
| 超級大獎 | 極低機率大量點數；解鎖「超幸運翻譯家」徽章 |

- 開發測試版可能使用 Google **測試廣告**，與正式上架後的 AdMob 單元不同
- 未完整觀看即關閉廣告：**不發放**點數

## 2. 點數消耗

| 項目 | 規則 |
|------|------|
| 翻譯 | **依 OCR 辨識字數計費** |
| 最低 | 3 點（短句） |
| 加價 | 每 50 字（含）多扣 1 點 |
| 扣點時機 | OCR 成功且翻譯完成後才扣點 |
| 點數不足 | 辨識後若點數不夠，不翻譯、不扣點 |

## 3. 點數保存

- 目前版本點數保存在**裝置本機**（AsyncStorage）
- 解除安裝、清除 App 資料可能導致點數歸零
- 正式商用版本計畫改為伺服器帳本驗證（尚未實作）

## 4. 異常與防濫用

以下情況我們可能暫停發點或調整餘額：

- 模擬器大量刷廣告
- 異常重複請求或疑似作弊
- 廣告平台回報無效流量

## 6. 規則變更

點數數值、爆擊機率或廣告單元可能因營運調整而變更，以 App 內最新顯示為準。
